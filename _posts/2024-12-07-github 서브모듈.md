# GitHub Submodule: 여러 프로젝트를 한 곳에서 관리해보자! 🎯

## Submodule? 이게 뭐냐고? 🤔
Git Submodule은 **다른 Git 리포지토리**를 지금 작업 중인 리포지토리 안으로 쏙 넣어서 관리할 수 있게 해주는 기능이야.  
즉, **메인 리포지토리 안에서 다른 프로젝트들을 독립적으로 관리**할 수 있는 거지.

### Submodule이 좋은 이유는 뭔데? 💡
- **여러 프로젝트**를 한 곳에서 간편하게 관리할 수 있음.
- 각 프로젝트는 **독립적인 버전 관리**가 가능.
- 코드 중복 없이 최신 버전을 쉽게 가져올 수 있음.

---

## 실습 목표: 메인 리포지토리에 팀 프로젝트 추가하기 🎮
우리가 할 일은 팀별 GitHub 리포지토리를 **메인 리포지토리**에 Submodule로 추가하는 거야.  
결과적으로 아래처럼 구성할 거라고 보면 돼:

```
Main Repository
├── 팀 1 리포지토리 (Submodule)
├── 팀 2 리포지토리 (Submodule)
├── 팀 3 리포지토리 (Submodule)
├── 팀 4 리포지토리 (Submodule)
└── 팀 5 리포지토리 (Submodule)
```

---

## 단계별 실습 🚀

### 1️⃣ Submodule 추가하기
각 팀의 리포지토리를 Submodule로 추가하자.

```bash
# 팀 1 리포지토리 추가
git submodule add https://github.com/team1/game-repo.git team1

# 팀 2~5 리포지토리도 추가
git submodule add https://github.com/team2/game-repo.git team2
git submodule add https://github.com/team3/game-repo.git team3
git submodule add https://github.com/team4/game-repo.git team4
git submodule add https://github.com/team5/game-repo.git team5
```

### 2️⃣ Submodule 초기화하고 업데이트하기
Submodule 추가 후 초기화와 업데이트는 필수!

```bash
# Submodule 초기화
git submodule init

# Submodule 업데이트
git submodule update
```

### 3️⃣ Submodule 상태 확인하기
제대로 추가되었는지 확인하려면 이 명령어를 쓰면 돼.

```bash
git submodule status
```

### 4️⃣ Submodule 변경사항 반영하기
팀에서 프로젝트를 업데이트했다면, 메인 리포지토리에서도 반영해줘야겠지?

```bash
# 특정 Submodule로 이동해서 업데이트
cd team1
git pull origin main

# 메인 리포지토리로 돌아와서 반영
cd ..
git commit -am "Updated team1 submodule"
```

---

## Submodule 수정사항 반영하기 🛠️

### Submodule 내부에서 수정하고 반영하기
1. **Submodule 디렉토리에서 수정**  
   Submodule 리포지토리로 이동해서 파일 수정 후 커밋 & 푸시.

```bash
# Submodule 디렉토리로 이동
cd team1

# 파일 수정 후 상태 확인
git status

# 변경 사항 커밋
git add <file>
git commit -m "수정 사항 반영"

# 원격 저장소에 푸시
git push origin main
```

2. **메인 리포지토리에서 업데이트**  
   Submodule에서 수정사항을 푸시한 후 메인 리포지토리에서 업데이트.

```bash
# 메인 리포지토리로 이동
cd ..

# Submodule 업데이트
git submodule update --remote team1

# Submodule 변경 사항 커밋
git add team1
git commit -m "Updated submodule team1 to latest version"
```

3. **메인 리포지토리 푸시**  
   Submodule 최신 버전을 반영한 후 메인 리포지토리 푸시.

```bash
# 메인 리포지토리 푸시
git push origin main
```

## Submodule 권한 문제 해결 방법 🔒

### 문제 상황
메인 리포지토리를 포크했을 때 Submodule의 읽기 권한이 없으면 문제가 생길 수 있어:
- Submodule 코드 접근 불가
- `git submodule init` 및 `git submodule update` 실행 시 오류 발생

### 해결책
1. **권한 요청**: Submodule 리포지토리에 대한 읽기 권한을 요청.
2. **대체 저장소 사용**: 접근 가능한 저장소로 Submodule을 대체.
